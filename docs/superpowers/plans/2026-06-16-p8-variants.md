# P8 — Variants Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development

**Goal:** Support morning/evening/weekly variants with different source sets and prompt templates, configurable via brief.yaml. Default variant: morning.

---

### Task 1: Define variant config model

**In brief.example.yaml:**
```yaml
output:
  variant: morning

variants:
  morning:
    sources: [weather, calendar, github, bahn, reddit, news]
  evening:
    sources: [weather, calendar, news, reddit]
  weekly:
    sources: [weather, calendar, github, news, reddit]
```

Update `BriefingConfig` to read `output.variant` and `variants.*` sections.

- [ ] Step 1: Add variant property to BriefingConfig
- [ ] Step 2: Write test → Commit

### Task 2: Thread variant through orchestrator

In `orchestrator.py`, `fetch_all` should filter enabled sources to those listed in the current variant:

```python
def fetch_all(config, variant: str = "morning"):
    enabled = config.enabled_sources()
    variant_sources = config.get_variant_sources(variant)
    if variant_sources is not None:
        enabled = [s for s in enabled if s.name in variant_sources]
    ...
```

If variant is not defined in config, fall back to all enabled sources (backward compat).

- [ ] Step 1: Write test that variant filters sources
- [ ] Step 2: Implement variant filtering
- [ ] Step 3: Commit

### Task 3: Create variant-specific prompt templates in locales

In locale YAML files, add:

```yaml
# en.yaml
prompts:
  morning:
    system_instruction: "..."
    structure: "Weather → Calendar → GitHub → ..."
  evening:
    system_instruction: "You are the Evening Briefing bot..."
    structure: "Weather (tomorrow) → Calendar (tomorrow) → ..."
  weekly:
    system_instruction: "You are the Weekly Briefing bot..."
    structure: "Weather (week ahead) → Calendar → GitHub → ..."
```

The `build_prompt` function selects the right prompt from locale based on variant.
